# ChefCraft GPT Wrapper on AWS Amplify

This is a cheap serverless starter for the ChefCraft recipe assistant using AWS Amplify Gen 2.

The app uses:

- Amplify Hosting for the static frontend, CDN, HTTPS, and custom domain
- Amplify Gen 2 backend for the Lambda function
- Amazon API Gateway HTTP API for the `/recipe` endpoint
- Amplify secrets for the OpenAI API key

The OpenAI API key lives only in Lambda. The browser calls the Amplify-managed `/recipe` API,
never OpenAI directly.

## Cheapest Practical Architecture

For a small public MVP, this avoids any always-on server. Amplify adds a little convenience cost,
but keeps deployment, HTTPS, branch previews, and custom domains simple.

- Domain: Route 53 registered domain, usually the largest fixed cost if you buy through AWS
- Amplify Hosting: usually low cost for small static apps
- API Gateway HTTP API: usage-based
- Lambda: usually free-tier covered for small traffic
- OpenAI: usage-based; start with `gpt-5-mini` for good recipe quality at low cost, or `gpt-5-nano` if you want absolute lowest spend

Set a hard monthly budget alert in AWS and an OpenAI project budget before sharing this publicly.

## Files

- `frontend/index.html` - the app shell
- `frontend/styles.css` - responsive styling
- `frontend/app.js` - browser logic
- `amplify/backend.ts` - Amplify Gen 2 backend definition
- `amplify/functions/recipe/handler.ts` - Lambda handler with the ChefCraft prompt
- `amplify/functions/recipe/resource.ts` - Lambda settings and OpenAI secret binding
- `amplify.yml` - Amplify Hosting build settings
- `template.yaml` and `scripts/deploy.ps1` - older non-Amplify deployment path, kept as a fallback

## Prerequisites

Install and configure:

- GitHub repository for this project
- An AWS account
- An OpenAI API key
- A domain in Route 53 or any registrar you can point to Amplify

## Local Install

```powershell
npm install
```

## Local Sandbox

```powershell
npm run amplify:sandbox
```

In another terminal, set the OpenAI key for the sandbox:

```powershell
npx ampx sandbox secret set OPENAI_API_KEY
```

Then run the frontend:

```powershell
npm run dev
```

## Production Deploy With Amplify

1. Push this project to GitHub.
2. Open the AWS Amplify console.
3. Choose **Create new app**.
4. Choose **Deploy with Git** and connect the repository.
5. Confirm the build settings. Amplify should detect `amplify.yml`.
6. Before the first successful production build, add the `OPENAI_API_KEY` secret in the Amplify app settings for the branch.
7. Deploy the branch.
8. In Amplify, open **Domain management** and connect your custom domain.

The frontend reads the generated `amplify_outputs.json` file at build time and calls the deployed
HTTP API endpoint directly.

## Local Frontend Preview

Use `npm run dev`. Opening `frontend/index.html` directly is no longer ideal because the production
API endpoint is generated by Amplify.

## Cost Controls

- Use `gpt-5-mini`; switch to `gpt-5-nano` only if cost matters more than nuance.
- Keep `max_output_tokens` conservative in `amplify/functions/recipe/handler.ts`.
- Set AWS Budgets and OpenAI project budgets before sharing the domain.
- Add Cognito auth, rate limiting, or a lightweight CAPTCHA before making it widely public.
